## 태그속성 class, id
### class 
* `<태그 class="값"></태그>
* 2번 이상 반복하는 이름인 경우
* 예) lnb, snb, container 등
### id 
* `<태그 id="값"></태그>
* 반복하면 안되는 단독 명칭인 경우
* 예) header, logo, gnb, footer 등
## HTML 기초 문서 태그
* 블록태그 :w100% 기본, 새로운 행, 크기가 큼
* 인라인태그 : 열(옆으로 나열), 크기는 내용만큼 인식
### 제목 
* `h1~h6` : 블록태그, h1 대제목, h6 소제목, h레벨이 높을수록(1에 가까울수록1~3) 검색어로 인식될 확률 높음. 
### 내용 
* `p` : 블록, 한줄~여러줄 단락형태의 내용일 때
* `br`: 인라인, 줄바꿈할 때 사용
* `em`: 인라인, 단순 문맥 강조, 주로 p자식으로 사용
* `strong` : 인라인, 중요+심각성 강조, p자식위주
* `blockquote` : 블록, 긴 인용문 
* `q` : 인라인, 짧은 인용문
* `code` : 인라인, 컴퓨터 명령어를 화면에 표시할 때
* `address` : 블록, footer안 주소 및 연락처, 해당 서비스 주소만 포함, address 안 블록 사용 주의(유효성 에러)
### 레이아웃
* `div` : 블록, 그룹, 자식이 2개 이상이며 같은 방향일 때, 블록과 인라인을 모두 가질 수 있음.
* `span`: 인라인, 그룹(위 div와 같음), 보조내용(디자인 보조, 포인트), 인라인만 자식으로 가질 수 있음. 
* `hr`  : 경계구분용(디자인 테두리 용도 아님) 
### 특수문자 태그
* `&copy;` : footer 저작권 마크   ⓒ
## 링크태그
* 필수구조 : `<a href="주소">클릭 대상</a>`
* 링크 새창 열기 속성 : `<a target="_blank">클릭대상</a>`
### 링크태그 href 주소 작성법
* 절대경로 : `href="https://주소"` 실제 서비스되는 웹주소
* 상대경로 : `href="./폴더 및 파일"` 같은 위치 또는 같은 위치
* 상대경로 : `href_"../폴더 및 파일"` 상위 위치
### 바로가기링크 (스크롤 이동하며 특정 위치로 가기)
1. `<a></a>` : 태그 준비
2. 1번 링크 클릭 시 이동하고 싶은 위이체 해당하는 태그에 `id="위치명"` 입력하기, 예) `<body id="top"></body>`
3. 1번 링크에 2번 아이디 연결하기 `<a href="#이동아이디명"></a>` **(#필수)**
## 이미지(img_) -inline
* `<img src="이미지경로" alt="대체텍스트">`
* `<img src="./naver_logo.png" alt="로고">`  (X)
* `<img src="./naver_logo.png" alt="네이버">`  (O)
* `<img src="./newneek_logo.png" alt="뉴닉">`  (O)
* `<img src="./cart.png" alt="장바구니 버튼">`  (X)
* `<img src="./cart.png" alt="장바구니 메뉴">`  (X)
* `<img src="./cart.png" alt="장바구니">`  (O)
* `<img src="./user.png" alt="로그인 하기">` (O)
* `<img src="./user.png" alt="로그인 ">` (O)
* `<img src="./user.png" alt="로그인버튼">` (X)
* `<img src="./user.png" alt="로그인아이콘">` (X)
### 이미지 필수 속성
* `src` : 이미지경로 (**상대**/절대경로)
* `alt` : 대체텍스트 (의미가 있는 경우 `alt="의미"` 없으면 따옴표 안을 비워둔다.)
### 이미지 선택 속성
* `class` , `id` (이미지 외 모든 태그 동일)
## background-image (CSS 배경이미지)
### img 태그가 아닌 background-image 처리하는 경우
* 형제, 자식, 자손 요소와 겹쳐져서 디자인 된 경우
* 쇼핑몰 상품 썸네일, 히어로 배너, 광고배너 등..
#### 태그에 배경이미지 적용하는 작성 순서
1. `<태그 style="">`
2. `<태그 style="background-image:;">`
3. `<태그 style="background-image:url(경로);">`
